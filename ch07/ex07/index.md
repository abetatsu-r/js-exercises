## 解答

計算量はO(nlogN);

```
T(1) = O(1) // 返すだけだから
T(n)
= 2T(n/2) // 半分に分けて再帰する
  + cn // 元の配列を分けたり詰めたりする作業がmergeのタイミングで発生でこれは長さの定数倍程度

展開していくと

T(n)
= 2T(n/2) + cn(cは定数)
= 2(2T(n/ 2^2) + c * (n/2)) + cn
= 2(2T(n/2^2)) + (cn + cn)

nがおおよそ2^kとすると、T(1)＝O(1)から,

T(n)
= cn(だいたいkくらい) + O(1)*2^k

k すなわち log2nなので
cn(log2n) + n

オーダー記法にのっとり
O(nlogn)
となる
```
